# Рекомендации по подготовке Markdown для RAG

Чтобы файлы хорошо chunk-ились и давали релевантные результаты, соблюдай эти правила.

## ✅ Заголовки

- Начинай каждый смысловой блок с заголовка `##` или `###`
- Не пропускай уровни (`#` → `###` без `##`)
- Заголовки должны быть осмысленными — по ним строится контекст

```md
## Раздел темы
### Подраздел
Текст подраздела ...
```

## ✅ Размер секций

Одна секция = один чанк (до 1200 символов). Если секция длиннее — она будет принудительно разбита.

- Оптимально: 200–800 символов
- Максимум: 1200 символов (настраивается в `rag-sources.yaml`)

## ✅ Списки

- Используй маркированные (`-`) или нумерованные (`1.`) списки
- Избегай вложенных списков глубже 2 уровней

## ✅ Таблицы

Допустимы, но не делай таблицы шире 5 столбцов и длиннее 20 строк — они могут не вместиться в один чанк.

## ✅ Код

```md
Блоки кода обрамляй тройными обратными кавычками с указанием языка:
\`\`\`python
print("hello")
\`\`\`
```

## ❌ Чего избегать

- HTML-разметки (будет воспринята как текст)
- Спецсимволов вроде `\{` без экранирования
- Секций без заголовков (упадут в дефолтный чанк)
- Файлов длиннее 100 КБ (лучше разбить на несколько)
